搭网页的简单总结

Hugo

选择模板；网页编译；修改域名；

在github上将该网页库pull到本地，由于后续在本地修改内容后需要重push到github，所以需要git操作。git相关的操作已经完全忘记了，不过可以总结一下今天见到的几个核心命令

git init # 初始化

git remote add name https://github/website # 链接到库

git add . # 添加修改

git commit -m name “xxx” # 备注修改

git push -u name branch # 推送修改

最后一步推送需要密钥，所以推送之前先执行

git remote set-url name https://@github.com//.git

Hexo

node

下载linux binary nodejs文件：https://nodejs.org/dist/v16.16.0/node-v16.16.0-linux-x64.tar.xz

解压后移动到目标文件夹

tar xvf node

mv node ~/node

建立软连接

ln -s ~/node/bin/node /usr/local/bin/node

ln -s ~/node/bin/npm /usr/local/bin/npm

修改环境变量

vim /etc/profile

在最后一行添加:

export PATH=$PATH:~/node/bin

保存后执行 source /etc/profile

输入 node -v ;npm -v 测试是否成功

Hexo安装及操作流程

  1. 安装hexo: npm install -g hexo

  2. 生成一个文件夹 blog

  3. hexo init

  4. hexo s # 本地预览

  5. 打开 “_config.yml”文件, 修改最后的

    depoly:

    ​ type: git

    ​ repository: github上ssh的地址

    ​ branch: main

  6. npm install hexo-deployer-git –save

  7. hexo g # 生成页面

  8. hexo d #上传github，这里也需要预先挂载好github库以及密钥

网页修改与修饰

  1. 使用命令 hexo new page name，在source文件下会产生名为name的文件，修改其下index.md,添加属性，如果name=categories， 则添加 type: “categories”

  2. 在source/_posts/里的文章抬头添加

    title: Your title

    date: 2017-05-26 12:12:57

    categories: - name_category

    标签页同理。

    另外主题可以独立设置，下载后放置在hexo根目录/themes/themename下， 并修改配置文件中主题选项为 themename。
